## 前言

Notebook LM是 Google 开发的一款以源码增强生成（Source Grounding）为核心的 AI 研究助手。不同于常规的聊天机器人（如 ChatGPT），NotebookLM 并不依赖泛泛的互联网数据，而是严格约束在你提供的资料库中进行思考。它的核心核心优势：精准、透明、多模态。

- **杜绝“幻觉”**：它的所有回答都必须基于你上传的 PDF、网页、YouTube 视频或 Google 简报。如果资料里没写，它就不会乱编。
- **透明溯源**：AI 的每一条结论都会附带**引用来源**。点击脚注，它会直接定位到源文件的具体段落，实现秒级查证。
- **多模态输出**：它不仅能和你聊天，还能把你的笔记一键转化为：
  - **语音摘要 (Audio Overview)**：支持 50 多种语言的 Podcast 级双人对谈。
  - **影片摘要 (Video Overview)**：自动生成带有旁白和视觉重点的演示视频。
  - **学习全家桶**：自动生成测验题、心智图、学习卡片和数据报表。

## 为什么NotebookLM 需要专属提示词

尽管 NotebookLM 天生就很聪明，但如果不给它设定**角色**和**逻辑框架**，它往往只会提供平铺直叙的总结。为了让它能像资深分析师一样犀利，或像幽默博主一样接地气，我们需要通过**结构化的提示词**来“唤醒”它深层的分析能力。今天，我将分享一套经过深度润色的**提示词模板**，帮你彻底榨干它的潜力。

## 全局提示词：构建你的知识基石

全局提示词（System Prompt）决定了 AI 对待你整个笔记本的态度。

## 六大功能模块自定义提示词

除了对话框，右侧工作区的6个功能模块（摘要、简报等）也可以独立设置提示词。
区别：系统提示词管整个笔记本（AI对话），模块提示词只管当前这个功能。

提示词通用底层逻辑：
1.目标与受众：给谁看？（给儿童看绘本，还是给投资人看财报？）
2.证据约束：必须严格基于资料，还是可以自行补全或使用训练数据（总结类任务一定要约束，防止幻觉）。

### 语音摘要

两个AI对话的语气、受众和讨论重点，避免它们闲聊无关内容。

- 指定受众身份：明确说明是讲给谁听（如初学者、CTO、投资人），决定内容的深浅。
- 设定讨论基调：指定对话是激烈的辩论、客观的分析还是轻松的科普。
- 圈定内容范围：明确指出需要忽略哪些细节（如代码实现），着重讨论哪些结论（如商业价值）。

```
## 示例1
受众是并没有技术背景的企业高管。请用商业分析的语气，重点讨论文献中提到的技术方案对降低成本的实际贡献。忽略所有具体的代码实现细节，最后总结出3个关键的决策建议。

## 示例2
设定两位主持人为‘激进派技术员’与‘保守派财务总监’。请他们针对文献中的技术方案进行 5 分钟的深度辩论。重点讨论 ROI（投资回报率）而非技术参数，语气要具有火药味，最后达成一个折中共识。
```


### 影片摘要
决定生成的视频画面视觉风格，以及视频脚本的叙事节奏。

- 描述视觉风格：详细描述想要的画面感觉（如极简线条、手绘风格、科技感），这是控制画面的关键。
- 规划叙事结构：指定视频的开头、中间和结尾分别展示什么内容。
- 设定应用场景：说明视频用于什么平台（如竖屏短视频、会议开场），这会影响画面的构图和节奏。

```
视觉风格采用深色背景的极简线条动画，强调科技感。叙事结构上，开篇直接展示旧方案的痛点，中间通过数据对比展示新方案的优势，结尾必须包含强有力的行动号召。适合在移动端竖屏播放。
```

### 学习卡

指定从文档中抽取哪些知识点，并规定卡片正面和背面分别写什么
- 限定考察范围：指定只从特定的章节或特定的主题（如某个理论）中提取内容。
- 定义正反面逻辑：明确正面是问题、场景还是名词，背面是答案、解释还是对应术语。
- 控制知识颗粒度：说明是考宏观概念的理解，还是考微观参数的记忆。

```
## 示例1
只针对素材中提到的 RAG技术架构 相关内容制作卡片。卡片正面描述一个具体的报错或应用场景，卡片背面
给出对应的解决方案或技术术语。不要制作简单的名词解释卡片，要侧重实战应用。

## 示例2
针对 [特定章节] 制作卡片。正面提供一个‘复杂故障场景描述’，背面提供‘排查步骤’及‘涉及的核心概念’。要求：背面内容需包含一个帮助记忆的口诀。
```



### 测验

设定出题的场景背景、难度陷阱以及考察的具体能力。

- 设定题目情境：将题目置入具体的工作场景中，而不是干巴巴的填空。
- 调整干扰项难度：要求生成的错误选项具有迷惑性，考验真正的理解程度。
- 明确考察目标：说明是为了复习概念定义，还是为了测试解决问题的能力。

```
生成一组基于真实工作场景的判断题。假设我是项目经理，根据文档内容，当我面临服务器负载过高时，哪种方案是当下的最优解？请提供带有强迷惑性干扰项的选择题，重点考察对性能优化的理解。
```



### 资讯图表

描述图表的视觉主题、配色方案以及需要强调的关键数据。

- 指定视觉主题：明确配色方案（如深色模式、企业蓝）和整体氛围
- 筛选核心数据：强制要求图表只展示最重要的3-5个关键指标，避免信息过载。
- 定义图表类型：说明是需要流程图、对比图还是时间轴，以及布局的方向（横向/纵向）。

```
制作一张横向的对比图表。左边列出传统工作流，右边列出AI自动化工作流。使用高亮颜色突出显示 节省日间和 降低成本 这两个关键数据指标。整体配色使用深蓝色调，风格要简洁专业。
```



### 简报PPT
 规划PPT的大纲结构、每一页的重点内容以及演讲对象的层级

- 控制幻灯片页数：强制限定生成的页数，防止内容过于冗长
- 构建逻辑链条：设计每一页的主题流向（如提出问题--分析原因--给出方案）
- 规范内容密度：要球每一页最少一个图片，不要纯文字堆砌，要适合演讲演示。

```
## 示例1
生成一份5页的融资演示文稿。第一页只讲用户痛点，第二页讲我们的解决方案，第三页展示商业模式。每页幻灯片字数严格控制在30字以内，主要使用列点形式，并在备注栏生成详细的演讲逐字稿。

## 示例2
生成 8 页路演 PPT 大纲。每页必须包含：1. 核心金句（15 字内）；2. 三个支撑要点；3. 建议配图说明（如：此处应展示增长趋势折线图）。风格要求：高冷、极简、商务感。
```
